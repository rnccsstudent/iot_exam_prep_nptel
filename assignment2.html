1)Based on functionality, MQTT is a _______ protocol.





MQTT (Message Queuing Telemetry Transport) is a lightweight transport protocol designed for low-bandwidth, high-latency, or unreliable networks. It enables communication between devices by transporting messages through a publish-subscribe model, making it ideal for IoT (Internet of Things) applications.

2)MQTT is designed for -





MQTT is specifically designed for remote connections (like sensors or devices far away) and limited bandwidth environments. It works efficiently even with low power and unreliable networks, which makes it perfect for IoT applications.

3)MQTT protocol follows _____________ paradigm for exchanging messages.





MQTT uses the publish-subscribe messaging paradigm, where clients can publish messages to a topic and others can subscribe to that topic to receive messages. This allows for decoupled communication between devices.

4)State True or False. Statement: ‚ÄúIn MQTT, the Subscribers are Lightweight Sensors.‚Äù



In MQTT, Subscribers are not necessarily lightweight sensors. Lightweight sensors are typically Publishers, sending data to the broker. Subscribers are clients (like apps, servers, or dashboards) that receive the data.

5)Which of the following is MQTT component?





In MQTT, the Broker acts as a middleman between publishers and subscribers. It receives messages from publishers and routes them to the appropriate subscribers based on the topic.

6)State True or False. A topic in MQTT can only be numbers.



In MQTT, a topic is a text string (not just numbers) used to organize and route messages. Topics can include letters, numbers, and special characters, and they are usually structured in a hierarchical format like home/temperature/livingroom.

7)State True or False. There are only two methods specified by the MQTT protocol.



MQTT defines multiple methods (called control packet types), not just two. Examples include:
CONNECT
PUBLISH
SUBSCRIBE
UNSUBSCRIBE
DISCONNECT
PINGREQ, etc.
These allow clients to connect, publish/receive messages, and manage communication with the broker.

8)The Publish/Subscribe architecture in MQTT is __________ driven.





The Publish/Subscribe architecture in MQTT is event-driven, meaning actions are triggered by events such as a sensor sending data or a client subscribing to a topic. This allows real-time, efficient communication between devices.

9)State True or False. The topic is the routing information for the broker.



In MQTT, the topic acts as routing information for the broker. When a publisher sends a message to a specific topic, the broker uses that topic to determine which subscribers should receive the message.

10)CoAP is _____________ and ____________.





CoAP (Constrained Application Protocol) is:
Based on HTTP, as it uses a similar request/response model.
Designed for M2M (Machine-to-Machine) applications, especially in constrained environments like IoT, where devices have limited power and bandwidth.

11)In CoAP, client-server interaction is asynchronous over a datagram transport protocol such as ____________.





In CoAP, the client-server interaction is asynchronous and typically uses UDP (User Datagram Protocol) as the transport protocol. UDP is chosen because it is lightweight and suitable for constrained devices and networks, which is common in IoT applications.

12)What is the full form of AMQP?





AMQP stands for Advanced Message Queuing Protocol, which is an open standard for message-oriented middleware that enables communication between systems using a queue-based model for messaging, ensuring reliable message delivery and handling.

13)AMQP has _______ number of frame types.





AMQP defines 6 frame types used for communication between message brokers and clients. These frame types handle various aspects of the communication, such as opening and closing connections, sending and receiving data, and handling errors.
